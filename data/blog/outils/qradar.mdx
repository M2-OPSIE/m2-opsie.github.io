# IBM QRadar : Le Moteur de Détection des Cyberattaques

## 1. Introduction

La sécurisation des systèmes d'information nécessite aujourd'hui des mécanismes avancés de détection, de corrélation et de réponse aux incidents. Les environnements hybrides (on-premise, cloud, SaaS) génèrent un volume massif de journaux et d'événements qu'il est indispensable de centraliser et d'analyser.

Dans ce contexte, IBM QRadar constitue une solution SIEM de référence permettant :

- La centralisation des événements de sécurité
- La corrélation en temps réel
- La priorisation des incidents
- L'aide à l'investigation
- L'intégration avec des plateformes d'automatisation (SOAR)

## 2. Présentation générale de la solution

### 2.1 Définition d'un SIEM

Un SIEM (Security Information and Event Management) est une plateforme qui :

- Collecte les logs
- Normalise les données
- Corrèle les événements
- Génère des alertes consolidées
- Fournit des capacités d'investigation

### 2.2 Positionnement de QRadar

IBM QRadar est conçu pour :

- Les SOC (Security Operations Centers)
- Les grandes entreprises
- Les environnements critiques
- Les architectures distribuées

La solution peut être déployée :

- Sur appliance physique certifiée
- En machine virtuelle
- En mode SaaS (QRadar on Cloud)

## 4. Fonctionnement interne du moteur SIEM

### 4.1 Collecte

Protocoles supportés :

- Syslog
- SNMP
- JDBC
- API REST
- Log File
- WinCollect (Windows)

### 4.2 Normalisation – DSM

Les Device Support Modules (DSM) assurent :

- L'identification du type de log
- L'extraction des champs normalisés
- La catégorisation (Log Source Type)

### 4.3 Corrélation

Le moteur applique :

- Règles basées sur conditions
- Corrélation multi-événements
- Seuils temporels
- Analyse comportementale

La corrélation produit une Offense.

### 4.4 Calcul de la magnitude

**Magnitude = fonction de :**

- Relevance
- Severity
- Credibility

## 5. Installation officielle (Processus académique)

⚠️ **QRadar doit être installé via ISO IBM officiel.**

### 5.1 Démarrage installation

Boot sur ISO :

- Install IBM QRadar

### 5.2 Configuration réseau

Après installation :

```bash
nmtui
```

Configurer :

- Adresse IP statique
- Gateway
- DNS
- Hostname

### 5.3 Vérification des services

```bash
systemctl status hostcontext
systemctl status ecs-ec
systemctl status tomcat
```

### 5.4 Redémarrage complet des services

```bash
service hostcontext restart
```

## 6. Ajout d'une source Syslog (exemple)

Configuration côté firewall :

```
logging host 192.168.10.50
logging trap informational
```

Dans QRadar :

- Admin → Log Sources → Add Log Source
- Type : Syslog
- Protocol : UDP/TCP

QRadar supporte :

- REST API
- Feeds externes

## 9. Intégration IP Reputation – Criminal IP

### 9.1 Principe

Criminal IP est une plateforme externe fournissant :

- Score de réputation IP
- Historique d'activités malveillantes
- Données OSINT

L'intégration se fait via API REST.

### 9.2 Exemple d'appel API

```bash
curl -X GET https://api.criminalip.io/v1/ip/8.8.8.8 \
-H "x-api-key: VOTRE_CLE_API"
```

### 9.3 Intégration dans QRadar

**Méthodes :**

- Script d'enrichissement
- Application App Exchange
- Intégration dans un playbook SOAR
